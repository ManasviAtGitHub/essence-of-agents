# 2026-07-05

## Enumerable short-codes

- **Ask:** a user reported they could reach other people's links by changing the code in the
  URL by one character (`/-/c` -> `/-/d`).
- **Summary:** codes were `base62(row_id)`, so they came out sequential (`b, c, d, ...`) and
  anyone could walk them.
- **Decision:** switch to random 7-char codes drawn from the base57 alphabet, collision-checked
  against the store. `make_code` loses its `row_id` argument entirely. Existing codes stay as
  they are (codes are permanent); only new ones are random.
- **Why:** guessable codes are a full authorization bypass -- the code *is* the capability.
  Randomness is the security model. Wrote it up as `lessons.md` #1 and a non-negotiable in
  CLAUDE.md; pinned the signature in `test_codec.py` so it can't quietly come back.
