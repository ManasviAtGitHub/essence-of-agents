# 2026-07-09

## Lost write under two creates

- **Ask:** a load test fired two `create`s back-to-back; afterward one of the two new links
  was simply gone from `runtime/snip.json`.
- **Summary:** `store.put` did read-modify-write on the whole file with no atomicity. The
  second call loaded the file before the first had finished writing, then overwrote it -- a
  classic lost update.
- **Decision:** `_save` now writes a temp file in the same dir and `os.replace`s it (atomic on
  POSIX + Windows), under a process lock. `put` goes through `_save`.
- **Why:** the store is the source of truth; a silently dropped link is the worst failure it
  can have. Recorded as `lessons.md` #2 and the write-temp-then-replace rule in the Do list;
  detail in `store.md`.
