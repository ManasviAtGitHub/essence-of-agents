Talk to compile - the natural-language compiler

What we are building: an LLM compiler. You type a request in plain English, and it produces a real, runnable plan - the same agent loop you built in course 1 - reliably, every time. A normal compiler turns code into a program by exact rules; here the "source" is a sentence and the reader is an AI model, which makes things up. So a grammar stands in front of the model and makes a wrong token impossible - and a verifier checks the result. That is the whole trick, and this is it happening.

How to read it: click a request -> its candidate tokens fly at the grammar gate, illegal ones bounce off, the legal winner snaps into the plan graph, then the verifier checks it. Grammar, plan and verifier run live here; the model's scores are illustrative until a real run is baked in.
