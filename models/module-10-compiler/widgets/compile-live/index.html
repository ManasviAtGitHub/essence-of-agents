Talk to compile - the natural-language compiler

What we are building: an LLM compiler. You type a request in plain English, and it produces a real, runnable plan - the same agent loop you built in course 1 - reliably, every time. A normal compiler turns code into a program by exact rules; here the "source" is a sentence and the reader is an AI model, which makes things up. So a set of rules (a "grammar") stands in front of the model and makes a wrong word impossible - and a checker (a "verifier") confirms the result before it runs. That is the whole trick, and this is it happening. (New to compilers? The four pieces are spelled out just below, and the animation introduces each one before it uses it.)

How to read it: click a request -> its candidate words fly at the rules (the gate), the disallowed ones bounce off, the winner snaps into the plan, then the checker confirms it. The rules, the plan and the checker run live here; the model's scores are illustrative until a real run is baked in.
